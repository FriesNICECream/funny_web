# UI Contract: Media Dashboard

## 1. 页面入口合同

- 已登录用户进入主应用后，必须能够访问“影视数据”主视图。
- 页面入口需要保留现有会话上下文，包括用户信息展示与退出入口。
- 当前视图的顶部导航项与侧边导航项必须同步呈现激活态。

## 2. 页面结构合同

页面最少应包含以下可见区域：

1. 头部区域
   - 页面标题
   - 简要说明
   - 搜索、筛选或排序入口
   - 通知或账户操作区
2. 导航区域
   - 顶部分类导航
   - 侧边主导航
3. 内容区域
   - 资源卡片网格
   - 每张卡片的元信息区
   - 每张卡片的链接区

## 3. 资源卡片合同

每张资源卡片必须满足：

- 展示封面图或明确占位
- 展示标题
- 展示至少一项补充元信息，例如年份、评分、标签或规格
- 展示资源链接区域
- 在悬浮、聚焦或点击时提供可识别交互反馈

## 4. 链接交互合同

- 每条资源链接必须具备可识别名称。
- 链接可用时，用户应能直接触发访问目标地址。
- 链接不可用时，界面应展示不可用状态，而不是保留无意义可点击元素。
- 当链接名称过长时，界面需要保持可读，不得挤压掉主要操作区域。

## 5. 筛选与反馈合同

- 搜索、筛选或排序至少提供两类可操作入口。
- 用户触发筛选后，页面必须反馈当前结果变化、激活条件或无结果状态。
- 当没有数据、没有匹配结果或字段缺失时，页面必须展示说明性反馈。

## 6. 响应式合同

- 桌面端默认展示完整导航与多列卡片布局。
- 移动端允许折叠侧边栏，但必须存在明确打开入口。
- 在移动端中，用户仍需能够完成“浏览资源 -> 使用筛选/导航 -> 点击链接”的主流程。
