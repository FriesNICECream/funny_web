# Quickstart: Stitch Media Data 仪表盘

## 1. 准备环境

1. 在项目根目录安装依赖：`pnpm install`
2. 启动开发环境：`pnpm dev`
3. 使用浏览器打开本地开发地址，并完成登录进入已登录态

## 2. 验证核心浏览流程

1. 进入已登录仪表盘后，确认当前主视图为“影视数据”或等价主内容区域。
2. 确认页面头部可见标题、副文案、搜索或筛选入口。
3. 确认资源卡片网格已渲染，且每张卡片至少包含封面、标题、评分、年份、标签和链接区域。
4. 点击任意资源链接入口，确认页面给出明确跳转或点击反馈。

## 3. 验证导航与筛选流程

1. 检查顶部导航与侧边导航的激活态是否与当前页面一致。
2. 输入关键词或触发筛选入口，确认页面结果区域发生可感知变化。
3. 切换排序或分类后，确认卡片区仍可浏览，且不会出现布局跳动到不可用的程度。
4. 清空条件后，确认页面可恢复默认内容视图。

## 4. 验证响应式与异常状态

1. 在桌面宽度下确认侧边栏、头部和卡片网格同时可见。
2. 将视口缩小到移动宽度，确认侧边栏默认折叠且可通过明确入口展开。
3. 验证长标题、长链接文案或缺省字段时，卡片仍保持可读和可操作。
4. 验证空列表或无结果状态时，页面会显示明确反馈，不会只剩空白区域。

## 5. 交付前检查

1. 运行构建命令：`pnpm build`
2. 复查桌面端与移动端截图，确认视觉层级、激活态和主操作入口一致。
3. 对照 `spec.md` 的成功标准逐项验收主流程。
